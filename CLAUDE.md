# 日々. — Language Practice / プロジェクト規約

個人運営の語学学習ツール集（静的サイト・GitHub Pages 公開）。データは全てブラウザの localStorage に保存し、サーバーは使わない。**作業するセッションはこの規約に必ず従うこと。**

## 更新履歴（Changelog）の書き方 ★最重要

トップページ `index.html` の `const updates = [...]` 配列がトップに表示される更新履歴。新ツールの公開・機能追加・修正をしたら、ここに**1件足す**。

- **必ず “1行スタイル”**：`text` は「**何をした — ひとことコメント**」の1行で簡潔に。長文・段階説明・箇条書きは禁止。
  - 良い例：`"Daily 일본어（JLPT N5）を公開 — 韓国語ユーザー向け、한일대조ノート付き。"`
  - 悪い例：仕様や対象文法を延々と列挙する数行の説明。
- **重複を増やさない**：同じツールの段階的な更新（例「TOPIK2まで対応」→「TOPIK3まで対応」…）を毎回追加しない。**既存の関連エントリを1件に書き直す/まとめる**。
- **最新が先頭**。`date: "YYYY-MM-DD"`。
- **3言語必須**：`text` と `tag` は `{ ja, ko, zh }`。`ko`/`zh` が無いと `ja` が表示される。
- `tag` は「ツール名 · 種別」の短いラベル（例 `"中文 · Talk"`, `"한국어 · Talk"`, `"全般"`, `"トップ"`, `"Reader · NEW"`）。
- 表示は最新5件＋「もっと見る」、14日以内は NEW バッジ、30日超は自動で非表示。
- 詳しい仕様は `index.html` の `updates` 配列直前のコメント参照。

## ツール作成規約（ファイル名・lang）

`clips.js` 冒頭のルールに従う。要点：
- ファイル名 `<学習言語>-<機能>[-<対象ユーザー>].html`（日本人向けはサフィックス無し、韓国人向け `-ko`、中国人向け `-zh`）。
- `<html lang>` は**対象ユーザーの母国語**（日本人向け `ja` / 韓国人向け `ko` / 中国人向け `zh`）。`clips.js` がこれを読んで UI を自動で母国語化する。

## Daily 系（作文ドリル）の共通UI

`chinese-practice.html` / `korean-practice.html` / `chinese-practice-ko.html` / `japanese-practice-ko.html` の4本は **共通スタイル `daily.css` を読み込む**（さわやかなアクア＋Manrope＋クリーン）。

- **UI を変えるときは `daily.css` を1か所直せば4本に反映**される。各HTMLにCSSを書き戻さない。
- 学習言語テキスト（`.zh` クラス）のフォントは `<html data-target="ja|zh|ko">` で出し分け。UI言語フォントは `<html lang>`。
- アクセント色は `--accent` 1変数。ツール種類ごとに替えてよい（Daily＝アクア）。
- 操作：採点ボタンはカード内の `힌트/스킵` 直下（`.btn-primary.card-check`）、「次へ／もう一度」は結果カード末尾。**Enter は改行のみ**（スマホの変換確定での誤作動防止）。固定下バーは使わない。
- レベルは HSK / TOPIK / JLPT。各ツール固有機能（簡体↔繁体トグル、ピンイン、ローマ字）は保持する。
- `daily.css` を更新したら各HTMLの `daily.css?v=N` のNを上げてキャッシュを更新。
