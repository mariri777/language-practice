# 日々. — 自分専用クラウド同期 のデプロイ手順

Cloudflare Workers + KV を使った、**あなた専用**の同期APIです。
秘密コードを入れた端末だけが同期し、他の訪問者は今まで通りローカルのみ（あなたの枠を使いません）。

## 1回だけやる準備

> Cloudflare の**無料アカウント**が必要です（未登録ならブラウザで作成）。

このフォルダ（`cloud/`）で順に実行します。

```sh
cd cloud

# 1) wrangler にログイン（ブラウザが開くので許可）
npx wrangler login

# 2) KV namespace を作成 → 出力された id をコピー
npx wrangler kv namespace create HIBI

# 3) wrangler.toml の "PASTE_KV_NAMESPACE_ID_HERE" を、2)の id に書き換える

# 4) デプロイ → 表示される URL（https://hibi-sync.XXXX.workers.dev）をコピー
npx wrangler deploy
```

## サイト側に URL を設定

リポジトリ直下の `cloud.js` の先頭にある

```js
var API = "https://hibi-sync.YOUR-SUBDOMAIN.workers.dev";
```

を、上の 4) で出た**あなたの URL** に書き換えて commit & push する。
（URL が未設定のままなら、クラウド同期は自動で無効になり、サイトは従来どおり動きます。）

## 使い方（各端末で最初の1回だけ）

1. サイトの「⤓ 同期」ページを開く
2. 「クラウド自動同期」で **コードを発行**（または既存コードを入力）して保存
3. 別の端末でも同じコードを入力 → 以降は**何もしなくても**自動で揃う

## 動作の概要

- 端末でデータが変わると、約2秒後に自動でクラウドへ保存（PUT）。
- ページを開くと、クラウドが新しければ取得して反映（必要なら1回だけ自動リロード）。
- サーバー時刻で新旧を判定するため、端末の時計ずれの影響を受けない。
- 競合（複数端末を同時にオフライン編集）は「後から保存した方が勝ち」。基本は1台ずつ使えば安全。

## 無料枠の目安

KV 無料枠 = 1日あたり 読み取り10万 / 書き込み1000 / 削除1000、保存1GB。
1人の学習なら大きく下回ります。超過しても課金ではなくその日の同期が止まるだけ。
